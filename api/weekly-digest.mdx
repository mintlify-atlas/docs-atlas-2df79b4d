## Endpoint

```
GET /api/weekly-digest
```

Returns a weekly digest of market regime activity, summarizing recent alerts and their implications. The digest aggregates regime alerts from the past week to identify trends and patterns.

## Response

<ResponseField name="generatedAt" type="string" required>
  ISO 8601 timestamp when the digest was generated
</ResponseField>

<ResponseField name="digest" type="object" required>
  Processed weekly digest data
  
  <Expandable title="digest properties">
    <ResponseField name="recentAlerts" type="array">
      Alerts from the past 7 days
    </ResponseField>
    
    <ResponseField name="primaryRegime" type="string">
      Most frequently occurring regime this week
    </ResponseField>
    
    <ResponseField name="regimeChanges" type="number">
      Count of regime transitions during the week
    </ResponseField>
    
    <ResponseField name="dominantReasons" type="array">
      Most common alert reason codes this week
    </ResponseField>
    
    <ResponseField name="weekOverWeekDelta" type="object">
      Change metrics compared to previous week
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="alertCount" type="number" required>
  Total number of regime alerts in storage
</ResponseField>

<ResponseField name="deliveryCount" type="number" required>
  Number of alert deliveries that have been sent
</ResponseField>

## Example Request

```bash
curl https://your-domain.com/api/weekly-digest
```

## Caching Behavior

- **Runtime**: Edge
- **Revalidation**: Dynamic (no static cache)
- Generated on-demand from current regime alert storage

## Weekly Window

The digest considers alerts from the past 7 days (168 hours) from the current timestamp. This rolling window ensures the digest always reflects recent activity.

## Use Cases

- Weekly team briefings on market conditions
- Regime stability monitoring
- Trend identification across multiple alerts
- Email/Slack digest automation
- Dashboard summary widgets
